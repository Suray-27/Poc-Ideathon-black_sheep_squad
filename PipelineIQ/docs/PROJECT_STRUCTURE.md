# Project Structure

PipelineIQ is an AI data-engineering assistant: a Python CLI/REPL that uses Google
Gemini to turn natural-language requests into executable SQLite ETL pipelines, with a
self-healing repair loop.

## Directory Layout

```
PipelineIQ/
├── src/                      # Application source code
│   ├── __init__.py
│   ├── app.py                # Main application + interactive REPL
│   └── setup_db.py           # Seeds the demo warehouse (users, orders) + products.csv
├── tests/                    # Tests (offline; no API key required)
│   ├── __init__.py
│   └── test_smoke.py         # Verifies setup_db builds the expected schema
├── data/                     # Runtime data — NOT committed (see .gitignore)
│   ├── source_warehouse.db   #   SQLite demo warehouse (regenerated by setup_db.py)
│   ├── pipeline_output.csv   #   Output of the last executed pipeline
│   ├── sources/              #   External source files scanned by the app (CSV/JSON/Excel)
│   └── pipelines/            #   Persisted pipeline definitions (JSON) for fix/change/list
├── docs/                     # Documentation
│   └── PROJECT_STRUCTURE.md
├── requirements.txt          # Python dependencies
├── .env.example              # Environment-variable template (no real secrets)
├── .env                      # Local secrets (gitignored) — your real GEMINI_API_KEY
├── .gitignore
└── README.md
```

## Directory Descriptions

- **src/** — all application code. `app.py` holds the REPL, request routing, plan
  generation, execution, and the self-healing repair loop. `setup_db.py` seeds demo
  data. Both resolve paths relative to the project root, so they work from any CWD.
- **tests/** — offline tests that target `setup_db` only (no Gemini call), so they run
  without credentials.
- **data/** — runtime artifacts (database, CSV output, source files, saved pipelines).
  All of it is gitignored and regenerable; nothing here is source.
- **docs/** — project documentation.

## Configuration

Configuration is loaded from a single `.env` file at the **project root**
(`PipelineIQ/.env`). There is no separate `config/` directory.

| Variable | Purpose | Default |
|----------|---------|---------|
| `GEMINI_API_KEY` | Google AI Studio key (starts with `AQ.`) | — (required) |
| `SOURCES_DIR` | Folder scanned for external source files | `data/sources` |
| `MAX_REPAIR_ATTEMPTS` | Self-healing retry limit on SQL errors | `3` |
| `DATABASE_PATH`, `DEBUG`, `LOG_LEVEL` | App settings | see `.env.example` |

## Getting Started

```bash
cd PipelineIQ
pip install -r requirements.txt

# Configure credentials
cp .env.example .env        # then edit .env and set a real GEMINI_API_KEY

# Seed the demo warehouse and a sample source file
python src/setup_db.py

# Launch the interactive assistant
python src/app.py
```

## Running Tests

```bash
cd PipelineIQ
python -m unittest discover -s tests        # or: python -m pytest tests/ -q
```
